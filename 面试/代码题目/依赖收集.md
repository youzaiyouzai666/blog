# 依赖收集

我们可以一起实现一个简化版的 `reactive` 响应式系统，仿照 Vue 3 的响应式原理，使用 `Proxy` + `effect` 实现依赖收集与响应更新。

下面会 **先讲设计逻辑，再写代码实现，最后逐步解释每个部分**，适合理解+自己拓展。

---

## 🧠 一、设计逻辑概览

我们要实现类似下面的用法：

```js
const state = reactive({ count: 0 });

effect(() => {
  console.log("count is:", state.count);
});

state.count++;
// => 自动触发 effect，打印新的 count
```

### 🧩 模块拆分

#### 1. `reactive(obj)`

- 返回一个被 Proxy 包裹的对象。
- 拦截 `get`/`set`，用于依赖收集和触发副作用。

#### 2. `effect(fn)`

- 注册一个“副作用函数”，当响应式数据变化时自动调用。
- 内部维护一个 `activeEffect` 全局变量，用于依赖收集。

#### 3. 依赖收集机制（Dep）

- 用数据结构存储谁依赖了谁（即：哪个对象的哪个属性被哪个 effect 用到了）。
- 变化时找到这些副作用函数并重新执行。

## 🔧 二、代码实现（简洁可读）

```js
let activeEffect = null; // 当前活跃的副作用函数

// 用于保存 target -> key -> effects 的映射
const targetMap = new WeakMap();

// 依赖收集
function track(target, key) {
  if (!activeEffect) return;

  let depsMap = targetMap.get(target);
  if (!depsMap) {
    depsMap = new Map();
    targetMap.set(target, depsMap);
  }

  let dep = depsMap.get(key);
  if (!dep) {
    dep = new Set();
    depsMap.set(key, dep);
  }

  dep.add(activeEffect);
}

// 触发副作用
function trigger(target, key) {
  const depsMap = targetMap.get(target);
  if (!depsMap) return;

  const dep = depsMap.get(key);
  if (dep) {
    dep.forEach((effectFn) => effectFn());
  }
}

// 注册副作用函数
function effect(fn) {
  activeEffect = fn;
  fn(); // 立即执行一次，收集依赖
  activeEffect = null;
}

// reactive 实现
function reactive(target) {
  return new Proxy(target, {
    get(obj, key, receiver) {
      const result = Reflect.get(obj, key, receiver);
      track(obj, key);
      return result;
    },
    set(obj, key, value, receiver) {
      const oldValue = obj[key];
      const result = Reflect.set(obj, key, value, receiver);
      if (oldValue !== value) {
        trigger(obj, key);
      }
      return result;
    },
  });
}
```

---

## 📌 三、设计重点详解

### 1. `effect` 机制

- 设置 `activeEffect` 是为了在 `get` 时知道“谁在用这个属性”。
- 使用完后要清除，避免污染下一个 effect。

### 2. `track` 依赖收集

- 使用 `WeakMap -> Map -> Set`：
  - `WeakMap`: 以响应对象为 key（不会阻止垃圾回收）。
  - `Map`: 属性名 -> 副作用函数集合。
  - `Set`: 避免重复收集同一个函数。

### 3. `trigger` 派发更新

- 属性值发生变化（set）时，找到对应依赖并执行。

### 4. 为什么用 Proxy

- Proxy 比 `Object.defineProperty` 更强大，可拦截数组、动态属性。
- 拦截 `get` 时进行 `track`，拦截 `set` 时进行 `trigger`。

## ✅ 四、使用示例测试

```js
const state = reactive({ count: 0 });

effect(() => {
  console.log("effect run:", state.count);
});

state.count++; // 应该触发 effect
state.count++; // 再次触发
```

输出：

```
effect run: 0
effect run: 1
effect run: 2
```

## 🧱 五、后续可以扩展的点（进阶）

1. **清理依赖**：避免多次收集相同 effect。
2. **嵌套 effect**：需要一个 effect 栈。
3. **调度器**：提供异步更新、批量更新（如 `queueMicrotask`）。
4. **只读、浅响应、深响应等特性**。
5. **计算属性（computed）** 实现。
