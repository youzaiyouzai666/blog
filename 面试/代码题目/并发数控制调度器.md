# 问题：

并发数控制调度器：实现一个 Scheduler，最多并发 2 个任务，且每个任务结束后下一个任务补上来继续执行。

我们来实现一个**并发数控制的调度器 `Scheduler`**，它可以控制最多同时运行两个任务，任务完成后继续调度队列中的任务。这个模式在网络请求、爬虫、任务池等场景中很有用。

---

## 🧠 一、设计思路详解

### 🌟 目标

- 控制任务的最大并发数为 **2**
- 使用者可以通过 `scheduler.add(task)` 添加任务，`task` 是一个返回 Promise 的函数
- 每当一个任务完成，就从队列中取出一个新任务继续执行
- 所有任务完成后，调度器不再运行

---

## 🏗️ 二、核心组件设计

我们分成以下几个关键点来设计：

### 1. **任务队列**

- 用一个数组保存等待执行的任务（`taskQueue`）

### 2. **当前正在执行的任务计数器**

- 通过变量 `runningCount` 来追踪当前正在执行的任务数量

### 3. **调度执行函数**

- 每当 `add()` 添加新任务，或者有任务完成后，会尝试触发执行任务的函数 `runNext()`。

### 4. **并发限制机制**

- 在 `runNext()` 中只有当 `runningCount < maxConcurrency` 且队列中还有任务时，才会执行下一个任务。

---

## 🧪 三、代码实现

```javascript
class Scheduler {
  constructor(maxConcurrency = 2) {
    this.maxConcurrency = maxConcurrency; // 最大并发数
    this.runningCount = 0; // 当前正在运行的任务数
    this.taskQueue = []; // 等待执行的任务队列
  }

  add(taskFn) {
    return new Promise((resolve, reject) => {
      // 将任务包一层，加入队列
      this.taskQueue.push(() => {
        return taskFn().then(resolve).catch(reject);
      });
      this.runNext(); // 尝试启动任务
    });
  }

  runNext() {
    // 如果已经到达最大并发或队列为空，就不做任何处理
    if (
      this.runningCount >= this.maxConcurrency ||
      this.taskQueue.length === 0
    ) {
      return;
    }

    const task = this.taskQueue.shift(); // 取出一个任务
    this.runningCount++; // 占用一个并发位置

    // 执行任务，并在结束后释放并发位置
    task().finally(() => {
      this.runningCount--;
      this.runNext(); // 再尝试下一个任务
    });
  }
}
```

## 🔁 四、使用示例

```javascript
// 注意返回是一个方法，两个箭头函数
const timeoutTask = (time, name) => () =>
  new Promise((resolve) => {
    console.log(`任务 ${name} 开始`);
    setTimeout(() => {
      console.log(`任务 ${name} 完成`);
      resolve();
    }, time);
  });

const scheduler = new Scheduler(2);

scheduler.add(timeoutTask(2000, "A"));
scheduler.add(timeoutTask(1000, "B"));
scheduler.add(timeoutTask(500, "C"));
scheduler.add(timeoutTask(3000, "D"));
```

---

## 🧩 输出顺序说明

输出将会是：

```
任务 A 开始
任务 B 开始
任务 B 完成
任务 C 开始
任务 A 完成
任务 D 开始
任务 C 完成
任务 D 完成
```

解释：

- A 和 B 同时开始（并发上限 2）
- B 比 A 早完成，C 被调度进来
- A 完成后，D 被调度进来
- 如此类推

## 📦 总结关键点

| 模块           | 作用                   |
| -------------- | ---------------------- |
| `taskQueue`    | 存放待执行的任务       |
| `runningCount` | 控制并发数             |
| `add()`        | 加入任务并尝试启动任务 |
| `runNext()`    | 核心调度逻辑           |

🧠 原理拆解
我们需要构建 3 个部分：

✅ createWorkerTask()：构建可注入任务函数的 Worker

✅ SchedulerWithWorker：调度器封装

✅ 测试代码

```
function createWorkerTask(name, time) {
  return () => new Promise((resolve, reject) => {
    const workerCode = `
      self.onmessage = function(e) {
        const { name, time } = e.data;
        console.log("开始任务:", name);
        setTimeout(() => {
          self.postMessage("完成任务: " + name);
          self.close();
        }, time);
      }
    `;

    const blob = new Blob([workerCode], { type: 'application/javascript' });
    const worker = new Worker(URL.createObjectURL(blob));

    worker.onmessage = (e) => {
      console.log(e.data);
      resolve(e.data);
    };

    worker.onerror = (e) => {
      reject(e.message);
    };

    worker.postMessage({ name, time });
  });
}

class SchedulerWithWorker {
  constructor(max = 2) {
    this.max = max;
    this.queue = [];
    this.activeCount = 0;
  }

  addTask(taskFn) {
    this.queue.push(taskFn);
    this.#schedule();
  }

  #schedule() {
    if (this.activeCount >= this.max || this.queue.length === 0) return;

    const task = this.queue.shift();
    if (typeof task !== 'function') {
      this.#schedule();
      return;
    }

    this.activeCount++;
    task().finally(() => {
      this.activeCount--;
      this.#schedule();
    });

    this.#schedule();
  }
}

const scheduler = new SchedulerWithWorker(2);

scheduler.addTask(createWorkerTask("任务1", 1000));
scheduler.addTask(createWorkerTask("任务2", 2000));
scheduler.addTask(createWorkerTask("任务3", 500));
scheduler.addTask(createWorkerTask("任务4", 800));

```
