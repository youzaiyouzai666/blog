react 是中运行时框架，在数据发生变化后，没有直接去操作 dom，而是生成一个新的所谓的虚拟 dom，它可以帮助我们解决跨平台和兼容性问题，并且通过 diff 算法得出最小的操作行为，这些全部都是在运行时来做的.(对比 vue|Svelte，重编译的框架)

具体渲染过程：

1. 调和阶段需要做两件事

   - 计算出目标 State 对应的虚拟 DOM 结构
   - 寻找「将虚拟 DOM 结构修改为目标虚拟 DOM 结构」的最优更新方案

2. 提交阶段

   - 将调和阶段记录的更新方案应用到 DOM 中
   - 调用暴露给开发者的钩子方法，如：useLayoutEffect 等

3. 渲染

流程核心问题：

- 特别是虚拟 DOM，运行时 CPU 与内存消耗会更高，
- IO 问题会更严重。

主要基于这两个核心问题，拆解一下 react 在调度渲染优化：
让组件的渲染 “可中断” 并且具有 “优先级”，其中包括几个不同的模块，他们各自负责不同的工作。
抽象出几个实体：
![Alt text](img/react_reder.png)

- Scheduler（调度器）—— 调度任务的优先级，高优任务优先进入 Reconciler
- Reconciler（协调器）—— 负责找出变化的组件（使用 Fiber 重构）
- Renderer（渲染器）—— 负责将变化的组件渲染到页面上

任务调度通过优先级、时间切片和可中断机制，将复杂的 UI 更新拆解为可管理的小任务，在浏览器每一帧的空闲时间内分批处理。

双缓冲机制是 React Fiber 架构的核心设计之一，主要用于实现 ‌ 无闪烁的异步渲染 ‌ 和 ‌ 渲染过程的可中断/恢复 ‌。其核心原理是通过维护两棵 Fiber 树，交替完成渲染工作。

# react 19 核心更新

下面先给出 React 19 的核心更新概览，然后按主题分别展开说明。React 19 在架构、并发、表单处理、服务端渲染等多方面都有显著增强，旨在简化开发流程并提升性能。

## 概览

React 19 于 2024 年 12 月 5 日正式发布，引入了 **Actions**（可在 transition 中直接使用 async 函数处理 pending、error、乐观更新等）、一系列全新 Hook（`useActionState`、`useFormStatus`、`useOptimistic`、`use` 等）、**React Server Components** 的稳定支持及 **Server Actions**，并对 hydration、Context、ref、资源预加载等底层机制进行了优化，还新增对文档元数据、样式表、异步脚本、自定义元素的原生支持。这些改动将之前需要手动管理的常见场景自动化，大幅提升开发者体验与应用性能。citeturn0search0turn0search10

## 1. Actions 与表单处理

### 1.1 新的 Actions 概念

- 在 React 19 之前，表单提交、数据变更需要手动管理 loading、error、optimistic update 等状态；
- React 19 引入了 **Actions**，允许在 `useTransition` 或组件内部直接使用 `async` 函数，自动处理 pending 状态、错误捕获和乐观更新，简化了常见的数据变更流程。citeturn0search0

### 1.2 相关新 Hook

- `useActionState`：跟踪 Action 的当前状态（pending、error、success）。citeturn0search2
- `useFormStatus`：专门用于表单提交状态管理，自动禁用按钮并展示错误。citeturn0search2
- `useOptimistic`：简化乐观更新逻辑，在 UI 更新和后端确认之间无缝切换。citeturn0search2

---

## 2. React Server Components 与 Server Actions

### 2.1 Server Components 稳定版

- Server Components 从实验走向稳定，可在服务端渲染组件并仅发送必要的 UI 更新至客户端，降低首屏 JS 体积，提升初始加载速度。citeturn0search2

### 2.2 Server Actions

- 配合 Server Components，Server Actions 允许在服务器端定义可直接在客户端调用的函数，自动管理状态与错误，消除传统 API 撰写与调用的样板代码。citeturn0search7

## 4. Hydration、Context 与 Ref 优化

### 4.1 更精细的 Hydration 差异化处理

- 提升 SSR/CSR 切换时的错误报告和自动修复能力，减少闪烁与不一致。citeturn0search2

### 4.2 Context API 改进

- 支持直接以 `<Context>` 而非 `<Context.Provider>` 形式传递 value，简化嵌套写法。citeturn0search2

### 4.3 Ref 清理与传递

- 新增对 ref 作为普通 prop 传递的支持；
- 提供 cleanup 函数以避免挂载/卸载阶段的内存泄漏。citeturn0search2

---

## 5. 资源预加载与文档元数据支持

### 5.1 文档元数据（Document Metadata）

- 原生支持在组件树中声明 `<head>` 内的元信息（title、meta 等），无需额外库。citeturn0search4

### 5.2 样式表与异步脚本

- 增强对 `<link rel="stylesheet">` 和 `<script async>` 的控制，可在组件级别声明并自动注入。citeturn0search5

### 5.3 资源预加载（Preloading）

- 新增 API 方便声明资源预加载（images、fonts、模块等），优化关键资源的加载顺序。citeturn0search5

---

## 6. 其他重要改动

- **移除弃用 API**：React 18 中的若干实验性警告升级为正式弃用，鼓励迁移到新方案。citeturn0search10
- **Custom Elements 支持**：可更顺畅地将 Web Components 与 React 混用。citeturn0search10
- **错误报告**：对 hydration、Server Components 中的错误堆栈更友好，可定位到源组件。citeturn0search4

---

以上即 React 19 的主要更新。新版本通过 Actions 与全新 Hook 自动化常见异步场景，稳定 Server Components，并在底层渲染、资源管理、API 一致性等方面做了全面打磨，帮助开发者更高效地构建现代 Web 应用。
